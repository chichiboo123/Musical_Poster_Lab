1. 단계별 포스터 미리보기 창 사이즈 일관화 + 요소 위치 고정
🔍 문제:
각 단계별 UI 요소가 달라짐에 따라 미리보기 영역 사이즈가 달라지고, 요소 배치도 밀림.

✅ 개선 방향:
포스터 미리보기 영역(poster-preview)의 사이즈를 CSS에서 고정하거나 비율 기반(aspect-ratio)으로 통일.

내부 요소는 비율 기반 좌표(%) 또는 CSS Transform의 scale() 기능을 활용하여 화면 크기 변화에도 동일한 위치 유지.

브라우저 리사이즈 시에도 배치가 유지되도록 JavaScript에서 컨테이너 크기 감지 후 재조정.

2. 포스터 미리보기 좌측 상단 파란 원 제거
🔍 문제:
포스터 미리보기 외부에 불필요한 장식용 원이 계속 표시됨.

✅ 개선 방향:
개발자 도구(F12)로 해당 원의 클래스명을 찾아내고 CSS에서 display: none 처리.

::before, ::after 등 pseudo-element로 들어온 것일 수 있으므로 이 역시 점검 필요.

단순히 좌측 상단 고정된 장식이라면, .poster-preview-container > div:first-child 조건 등으로 필터링해 제거 가능.

3. 텍스트 및 이미지 요소의 가로 중심 정렬 미작동
🔍 문제:
가운데 정렬 버튼을 눌러도 요소가 정확히 가로 중앙으로 이동하지 않음.

✅ 개선 방향:
요소의 left 좌표를 조정하여, 포스터 미리보기 영역의 가로 중심에 배치되도록 처리.

가운데 정렬은 X축만 조정하고, Y축(top)은 유지해야 하므로 세로 위치가 변경되지 않도록 주의.

요소 중심이 아닌 좌측 기준으로 잡힌 경우, 요소 너비를 기준으로 중앙 오차 발생하니 너비 계산 반영 필요.

4. ‘처음으로’ 버튼 색상 문제
🔍 문제:
버튼이 흰 배경에 흰 글씨로 보이지 않음.

✅ 개선 방향:
배경색 또는 텍스트 색을 조정해 대비 확보.

마우스 오버 시 색상 변화로 사용자 인식 가능하게 UX 개선.

버튼 위치나 크기도 사용자 눈에 잘 띄는 곳으로 재배치 고려.

5. 공연정보 ‘한 번에 삽입’ 시 텍스트가 겹침
🔍 문제:
여러 줄의 정보가 한 위치에 덮여 보임.

✅ 개선 방향:
각 정보 항목마다 고정된 간격(top offset)을 두고 순차적으로 배치.

마치 줄바꿈한 것처럼 각 요소가 일정한 Y축 위치에 배치되도록 설정.

폰트 크기와 요소 간 간격도 UI 기준에 맞춰 조정.

6. PNG, PDF, 클립보드 복사 기능 미작동
🔍 문제:
다운로드 및 복사 버튼을 눌러도 반응 없음.

✅ 개선 방향:
html2canvas(화면 캡처) + jsPDF(PDF 생성) 라이브러리 활용하여 각 기능 연결.

복사 기능은 캔버스를 이미지로 변환한 뒤, 클립보드에 이미지 형태로 복사하는 로직 필요.

에러 발생 시 alert로 사용자 피드백 제공.

7. ‘처음으로’ 버튼 클릭 시 시작 페이지로 이동 안 됨
🔍 문제:
버튼을 눌러도 초기 화면(가로/세로 선택 화면)으로 이동하지 않음.

✅ 개선 방향:
버튼 클릭 시 단순 페이지 이동(window.location.href = 'start.html') 기능 연결.

사용자 상태 초기화도 함께 이루어지도록 전체 reset 함수 또는 상태 초기화 코드 삽입 필요.

8. 타이틀 앞 원 아이콘 제거 및 이모지로 교체
🔍 문제:
타이틀 앞의 주황색 원은 의미 없는 장식.

✅ 개선 방향:
타이틀 로고 영역에서 원형 span이나 배경색 스타일 삭제.

이 자리에 새로운 이모지(🖼️)를 삽입하여 직관적이고 재치 있는 디자인으로 개선.

시작 화면(가로/세로 선택)에서도 동일한 디자인 적용 필요.
